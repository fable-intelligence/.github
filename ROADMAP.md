# Roadmap

## Phase 0: Formation

- Publish engineering handbook v0
- Standardize labels, issue templates, and ADR format
- Define confidential-document handling rules

## Phase 1: Proof

- Connect this repo to the Fable simulator or reference prototype.
- Define launch-blocking tests and acceptance criteria.
- Produce a short weekly decision memo.

## Phase 2: Hardware Readiness

- Promote only validated capabilities into hardware requirements.
- Freeze interfaces only after simulator and design partner feedback.
- Keep regulatory, privacy, operator, and support needs visible.
